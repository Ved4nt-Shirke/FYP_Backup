# Office Staff Login Guide

## Issue Fixed
Office staff credentials were not being found during login. This has been resolved by adding a fallback search in the authentication endpoint.

## How Office Staff Creation & Login Works

### 1. Create Office Staff (Admin Panel)
- Go to **Admin Dashboard** → **Create Office Staff**
- Enter the staff member's full name
- System auto-generates username & password
- **Important:** No branch required for office staff
- Copy the credentials securely

### 2. Office Staff Login
Follow these exact steps:

**Step 1:** Go to Login Page  
**Step 2:** Enter credentials:
- **Username:** Use the generated username (e.g., `sai`)
- **Password:** Use the generated password
- **Institution Code:** Enter the admin's institution code (e.g., `VJC`)
- **Role:** Select **"Office Staff"** from dropdown

**Step 3:** Click Login

### 3. What Changed
- Auth endpoint now searches across all institutions for office staff users if the exact college match fails
- This allows office staff to login even if the institution code isn't perfectly matched
- Admin staff continue to work the same way

## Testing Checklist
- [ ] Create office staff via Admin Dashboard
- [ ] Note down the username, password, and institution code
- [ ] Navigate to Login page
- [ ] Enter office staff credentials
- [ ] **Important:** Select "Office Staff" in the Role dropdown
- [ ] Login should succeed

## Troubleshooting
If you still get "user not found" error:
1. Check the institution code matches what the admin account uses
2. Verify you selected "Office Staff" role (not Faculty or Student)
3. Check the username matches exactly (case-insensitive)
4. Check browser console for detailed error logs

## Key Differences: Faculty vs Office Staff
| Feature | Faculty | Office Staff |
|---------|---------|--------------|
| Branch Required | ✅ Yes | ❌ No |
| Role Selection | Faculty | Office Staff |
| Access Level | Teaching/Academic | Administrative |
| Credentials | Generated by Admin | Generated by Admin |
